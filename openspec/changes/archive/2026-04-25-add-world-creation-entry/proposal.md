## Why

织世录当前已经有“已有世界”的水墨工作台，但还缺少用户进入产品的第一个核心动作：创建一个新世界。现在补齐独立创建世界入口，可以把世界设定录入从工作台展示中拆出来，为后续角色生成、沙盘运行和叙事整理建立清晰起点。

## What Changes

- 新增独立的创建世界页面，路径建议为 `/worlds/new`。
- 在现有左侧导航中增加“新建世界”入口，用户可从工作台进入创建页。
- 创建页采用单页分区表单，不使用多步向导：基础设定、世界底色、运行种子。
- 表单首版包含六个字段：世界名称、世界类型、背景简介、叙事风格、世界规则、初始冲突。
- 世界类型使用选择卡而非普通下拉，内置“宫廷权谋、魔法学院、末日避难所、小镇悬疑、修仙宗门、自定义”六个选项。
- 当用户选择“自定义”类型时，页面展开自定义类型名称输入。
- 页面右侧实时展示“世界雏形预览”，反映名称、类型、叙事风格和初始冲突摘要。
- 点击“创建世界”后，首版不接后端，使用前端 mock 或本地状态进入现有工作台，并用创建信息替换工作台顶部世界信息。
- 暂不实现角色生成、后端持久化、数据库写入、Agent 世界运行、真实导航权限或登录流程。

## Capabilities

### New Capabilities

- `world-creation-entry`: 定义创建世界独立页面、入口导航、世界类型选择、表单字段、实时预览和首版创建后进入工作台的行为。

### Modified Capabilities

- `ink-dashboard-ui`: 左侧导航需新增“新建世界”入口，并允许工作台顶部世界信息在创建后使用前端 mock 或本地状态展示新世界基本信息。

## Impact

- 受影响代码：`src/app/page.tsx`、新增 `src/app/worlds/new/page.tsx`、可能新增 `src/components/world-creation/**`、`src/lib/**`、`src/types/**`，以及现有 dashboard 组件、mock 数据和全局样式。
- 受影响规格：新增 `world-creation-entry` 规格，并为 `ink-dashboard-ui` 增加导航入口与创建后展示行为。
- API 影响：无新增公共 API，无后端接口变更。
- 数据影响：首版仅使用前端 mock 或本地状态，不新增数据库表和持久化模型。
- 依赖影响：优先复用现有 Next.js、TypeScript、Tailwind CSS、shadcn/ui 和 lucide-react，不引入新的运行时依赖。
- 现有功能影响：现有工作台仍可作为默认已创建世界视图访问；新增入口不应破坏工作台三栏布局、mock 数据展示和响应式可读性。
- 回滚计划：删除本变更新增页面、组件、类型和样式，移除左侧“新建世界”入口及创建后本地状态接入，恢复 dashboard mock 数据为固定“烟雨江南”。
- 需要协调的团队：前端负责页面与交互实现；产品/设计确认字段文案、类型选项和视觉层级；后端/Agent 团队本阶段仅需知情，后续持久化和世界运行接入时再参与接口设计。
