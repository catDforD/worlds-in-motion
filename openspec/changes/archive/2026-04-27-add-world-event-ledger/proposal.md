## Why

当前工作台中的“近期事件 / 最新章节 / 运行状态 / 当前时间”仍以纯 mock 内容呈现，用户无法在本地维护世界已经发生过什么、当前推进到哪一天、是否暂停运行。首版需要在不接后端、不触发 Agent 的前提下，把这些模块升级为可持久化、可手动维护的世界运行记录，为后续真实世界运行和叙事生成提供前端契约。

## What Changes

- 新增本地 `WorldRuntimeState`，记录当前世界时间、运行天数、暂停状态、事件列表、章节草稿和最新章节。
- 工作台“近期事件”“最新故事章节”“当前时间”“运行状态”优先读取本地运行记录；没有本地记录时保留现有可读默认展示。
- “暂停运行”按钮从静态按钮变为真实切换暂停/运行状态，并将结果保存到本地。
- 新增“推进一日/记录事件”的轻量入口，允许用户手动推进世界日期并创建运行事件。
- 继续保持纯前端实现，不调用后端接口、不写入数据库、不触发 Agent 或自动生成流程。
- 回滚计划：如交互或本地持久化出现问题，可删除新增本地运行态读写与手动入口，使工作台回退到现有 mock 展示和静态按钮。

## Capabilities

### New Capabilities
- `world-event-ledger`: 定义本地世界运行记录的数据、持久化、手动推进一日、记录事件、暂停状态和章节记录行为。

### Modified Capabilities
- `ink-dashboard-ui`: 工作台运行态相关模块从纯 mock 展示改为优先读取本地世界运行记录，并在缺失本地记录时保持可读回退。

## Impact

- 影响前端工作台页面、运行态展示模块、本地存储工具和相关 TypeScript 类型。
- 不新增后端 API，不改变现有公共 API，不引入数据库迁移。
- 需要保持现有创建世界、本地内容种子、角色/关系/统计展示行为不被运行记录逻辑覆盖。
- 需要协调的团队：前端团队负责实现与验证；产品/叙事设计确认手动事件字段和章节草稿字段；后端/Agent 团队无需本期接入，但需知晓该本地状态契约后续可能升级为服务端运行记录。
