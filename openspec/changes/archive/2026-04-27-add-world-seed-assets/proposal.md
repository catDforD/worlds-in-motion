## Why

创建世界后，用户目前只能看到基础设定被带入工作台，角色、势力、地点和关系仍主要停留在静态展示，世界还没有形成“属于我”的可操作内容。先补齐浏览器本地的世界内容种子，可以在不引入后端、数据库或 Agent 编排的前提下，让创建后的世界具备可编辑、可恢复、可联动展示的内容资产。

## What Changes

- 新增世界内容种子管理能力，支持在当前世界下维护角色种子、势力种子、地点种子和关系种子。
- 每类种子提供首版必要字段：角色包含姓名、身份、目标、状态；势力包含名称、立场、资源、冲突点；地点包含名称、类型、重要性；关系包含 A 与 B 的关系、紧张度、备注。
- 将世界内容种子保存到 `localStorage`，刷新后可恢复；本地数据损坏时回退到可读默认状态。
- 让工作台中的角色、紧张关系和统计概览优先读取用户自己的本地种子数据。
- 保持首版纯前端实现，不调用后端接口、不写入数据库、不触发 Agent 或 LangGraph 流程。

## Capabilities

### New Capabilities
- `world-seed-assets`: 定义当前世界下角色、势力、地点和关系种子的本地创建、编辑、删除、保存和恢复能力。

### Modified Capabilities
- `ink-dashboard-ui`: 工作台角色、关系和统计项从静态 mock 展示扩展为优先读取当前世界的本地内容种子。

## Impact

- 受影响代码：Next.js 前端的工作台页面、世界状态/本地存储工具、可能新增的内容种子编辑组件或页面区域。
- 受影响数据：新增浏览器本地存储结构；需与现有创建世界临时状态、世界设定本地状态保持兼容。
- 受影响功能：工作台默认 mock 展示仍作为空数据或损坏数据时的回退；创建世界、世界设定编辑和导航不应发生破坏性变化。
- 不涉及公共 API、数据库 schema、后端服务、Agent 编排或外部依赖变更。
- 回滚计划：移除内容种子编辑入口与读取逻辑，清理新增本地存储 key 的使用点，工作台恢复到现有 mock/创建世界临时状态展示；已有浏览器本地种子数据可保留但不再读取。
- 需要协调的团队：前端产品与设计确认种子字段和工作台呈现优先级；后端/Agent 团队仅需知悉本阶段不接入服务端持久化和运行编排。
