## Why

当前工作台已经能展示默认世界或创建后的世界基本信息，但用户无法在进入工作台后继续维护世界背景、规则、风格边界、禁止事项和核心矛盾。世界设定是后续角色生成、沙盘运行和叙事整理的上游约束，现在需要先补齐纯前端编辑闭环，为后端世界模型出现前提供可验证的产品体验。

## What Changes

- 在工作台中提供“世界设定”编辑入口，允许用户打开当前世界设定编辑界面。
- 支持编辑并保存五类核心设定：世界背景、世界规则、风格偏好、禁止事项、核心矛盾。
- 将当前世界设定保存到浏览器本地状态，刷新后仍可恢复；本阶段不调用后端接口、不写入数据库、不触发 Agent 世界运行。
- 保存后工作台顶部世界横幅立即反映可展示的设定变化，例如世界背景和风格标签。
- 兼容现有创建世界流程产生的前端状态，让创建后的世界可以继续进入同一套设定编辑体验。
- 保持现有水墨工作台视觉风格，并保证桌面与窄屏视口下编辑界面可读、可操作。

## Capabilities

### New Capabilities

- `world-settings-editor`: 定义当前世界设定的前端编辑、保存、恢复和工作台展示联动能力。

### Modified Capabilities

- 无。现有 `world-creation-entry` 与 `ink-dashboard-ui` 的既有行为保持兼容，本变更通过新增能力扩展当前世界设定编辑契约。

## Impact

- 影响代码：
  - `src/components/dashboard/ink-dashboard.tsx`：接入“世界设定”入口、读取本地设定状态并更新顶部横幅。
  - `src/components/world-creation/create-world-page.tsx`：创建成功后可初始化新的本地世界设定状态。
  - `src/lib/world-creation.ts` 或新增 `src/lib/world-settings.ts`：抽取本地存储、解析、迁移和订阅逻辑。
  - `src/types/world-creation.ts` 或新增 `src/types/world-settings.ts`：定义可编辑世界设定类型。
  - `src/app/globals.css`：增加设定编辑面板、长文本字段和响应式样式。
- API 与依赖：
  - 不新增后端 API、数据库迁移、外部服务或 npm 依赖。
  - 不改变公共 API。
- 对现有功能的影响：
  - 默认工作台仍可在没有本地设定时显示“烟雨江南”mock 数据。
  - 创建世界入口仍保留 `/worlds/new` 独立页面和既有必填校验。
  - 工作台“世界设定”按钮从静态按钮变为可操作入口。
- 需要协调的团队：
  - 前端：实现编辑面板、本地状态和响应式验证。
  - 产品/叙事设计：确认字段文案、占位语和禁止事项的产品语义。
  - 后端/Agent：本阶段无需实现，但需知晓本地设定模型未来会迁移为正式世界模型输入。
- 回滚计划：
  - 移除新增的设定编辑组件、本地设定 helper 和相关样式。
  - 将工作台“世界设定”按钮恢复为无状态展示按钮。
  - 创建页恢复为写入原有临时创建状态，工作台继续只读取现有 mock 或创建后的基本信息。
