## Why

当前侧边栏的运行入口在用户点击“运行一天”后反馈不够明确，用户只能看到日期或运行天数变化，难以理解这一轮世界推进带来了哪些叙事变化。需要新增一个本轮结果面板，把事件、关系、秘密、角色目标和故事草稿等关键变化用可扫描的摘要呈现出来，让世界运行的产品体验更具体。

## What Changes

- 在侧边栏运行界面新增“本轮变化”结果面板，用户每次点击“运行一天”后立即展示本轮摘要。
- 结果面板展示本轮发生事件数、关系变化数、秘密发现数、角色目标改变数和故事草稿生成数，并可展示对应的简短明细。
- 将现有“推进一日”语义统一到用户可见的“运行一天”入口；首版仍限定为前端本地运行，不调用后端、不触发真实 Agent。
- 扩展本地 `WorldRuntimeState`，保存最近一次运行结果摘要，刷新或切换视图后仍能恢复 active world 的最近结果。
- 无本轮结果时展示可读空状态；运行结果字段缺失或旧本地数据损坏时必须安全回退。
- 回滚计划：移除结果面板和最近运行结果字段读取逻辑，保留原有“推进一日”日期与运行天数更新能力；旧本地记录缺少新增字段时不会影响现有工作台展示。

## Capabilities

### New Capabilities
无

### Modified Capabilities
- `ink-dashboard-ui`: 工作台侧边栏运行界面新增“运行一天”后的本轮变化结果面板展示要求。
- `world-event-ledger`: 本地世界运行记录扩展最近一次运行结果摘要，用于保存和恢复本轮变化统计与明细。

## Impact

- 影响代码：
  - `src/types/world-runtime.ts`：新增本轮运行结果摘要相关类型。
  - `src/lib/world-runtime.ts`：扩展本地运行记录解析、默认值、兼容旧数据和“运行一天”更新逻辑。
  - `src/components/dashboard/ink-dashboard.tsx`：调整侧边栏运行入口文案和交互，新增结果面板 UI。
  - 可能影响 `src/types/dashboard.ts` 与 `src/lib/dashboard-data.ts` 中用于工作台展示的派生数据。
- API 影响：无新增公共 API，无后端接口变更。
- 数据影响：浏览器本地运行记录会新增可选的最近运行结果字段；旧记录必须继续可解析。
- 依赖影响：优先复用现有 Next.js、TypeScript、Tailwind CSS、shadcn/ui 和 lucide-react，不新增运行时依赖。
- 对现有功能的影响：创建世界、世界切换、事件日志、手动记录事件、暂停/继续运行和现有 mock 回退应继续可用；结果面板不得覆盖近期事件列表或完整事件日志。
- 需要协调的团队：前端团队负责状态、交互和视觉实现；产品/叙事设计确认结果面板的分类命名、数字口径和明细文案；后端/Agent 团队本期暂无接入依赖，但需要知晓该本地结果结构后续可能升级为真实运行输出。
