## ADDED Requirements

### Requirement: 运行一天结果面板
系统 SHALL 在工作台侧边栏展示最近一次“运行一天”的本轮变化结果面板，用于呈现事件、关系、秘密、角色目标和故事草稿五类运行摘要。

#### Scenario: 初始无运行结果时显示空状态
- **GIVEN** active world 没有最近一次运行结果
- **WHEN** 用户访问工作台
- **THEN** 系统 MUST 在侧边栏结果面板显示可读空状态，并且 MUST NOT 显示伪造的非零统计

#### Scenario: 点击运行一天后展示本轮统计
- **GIVEN** 工作台已加载 active world 的本地运行记录
- **WHEN** 用户点击“运行一天”
- **THEN** 系统 MUST 更新当前世界时间和运行天数，并在结果面板展示本轮发生事件数、关系变化数、秘密发现数、角色目标改变数和故事草稿生成数

#### Scenario: 结果面板使用明确分类文案
- **GIVEN** 最近一次运行结果包含五类统计
- **WHEN** 结果面板渲染完成
- **THEN** 系统 MUST 使用类似“今日发生了 5 件事”“2 段关系发生变化”“1 个秘密被发现”“1 个角色目标改变”“生成了 1 段故事草稿”的分类文案展示结果

#### Scenario: 展示本轮变化明细
- **GIVEN** 最近一次运行结果中某个分类包含明细文本
- **WHEN** 用户查看结果面板
- **THEN** 系统 MUST 展示该分类的简短明细，并限制长文本展示，避免撑破侧边栏布局

#### Scenario: 刷新后恢复最近运行结果
- **GIVEN** 用户已经点击“运行一天”并保存了最近一次运行结果
- **WHEN** 用户刷新页面或重新访问工作台
- **THEN** 系统 MUST 从 active world 的本地运行记录恢复并展示最近一次运行结果

#### Scenario: 切换世界后展示对应结果
- **GIVEN** 两个世界分别保存了不同的最近运行结果
- **WHEN** 用户切换 active world
- **THEN** 系统 MUST 在结果面板展示切换后世界对应的最近运行结果

#### Scenario: 旧数据缺少结果字段时安全回退
- **GIVEN** active world 的本地运行记录来自旧版本且缺少最近运行结果字段
- **WHEN** 工作台渲染侧边栏结果面板
- **THEN** 系统 MUST 显示空状态或可读回退，并且 MUST NOT 让工作台渲染失败

#### Scenario: 窄屏端结果面板可读
- **GIVEN** 用户使用窄屏视口访问工作台
- **WHEN** 结果面板渲染完成
- **THEN** 系统 MUST 保持统计、明细、按钮和相邻模块可读，且不得出现明显遮挡或横向溢出

#### Scenario: 运行一天不调用后端
- **GIVEN** 用户点击“运行一天”
- **WHEN** 系统更新结果面板
- **THEN** 系统 MUST NOT 调用 FastAPI、数据库接口、远程持久化接口或 Agent 工作流
