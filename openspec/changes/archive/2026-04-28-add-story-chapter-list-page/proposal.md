## Why

当前工作台只展示少量最新故事内容，用户缺少一个集中浏览已生成章节的页面，无法快速回看每章生成时间、对应世界时间、主要角色和摘要。故事章节列表页是从“运行日志”走向可阅读故事产物的第一步，先用 mock 章节验证信息结构和阅读入口。

## What Changes

- 新增故事章节列表页，展示已生成故事章节的列表。
- 每个章节条目展示章节标题、生成时间、对应世界时间、主要角色、摘要和阅读按钮。
- 首版使用前端 mock 故事章节数据，不接入后端持久化，不触发 Agent 或故事生成流程。
- 提供章节阅读入口的前端占位行为，允许用户从列表进入或查看单个章节内容的可读状态。
- 保持现有水墨工作台视觉语言，并确保桌面端和窄屏端均可读。

## Capabilities

### New Capabilities
- `story-chapter-list-page`: 定义故事章节列表页的路由、mock 数据展示、章节元信息、阅读入口、空状态和首版本地边界。

### Modified Capabilities
- 无。

## Impact

- 影响代码：
  - 新增故事章节列表页相关路由和组件，例如 `src/app/stories/page.tsx` 或等价路径。
  - 可新增前端 mock 章节数据或轻量数据映射工具，用于集中管理章节标题、生成时间、世界时间、主要角色、摘要和正文占位。
  - 如需承接阅读按钮，可新增章节阅读占位视图或详情路由，例如 `src/app/stories/[chapterId]/page.tsx`。
  - 可按需补充全局样式或局部样式，保持现有宣纸、墨色、细线卡片和低饱和视觉语言。
- API 影响：无新增公共 API，无后端接口变更。
- 数据影响：首版只使用 mock 数据，不读取或写入浏览器本地运行记录，不改变现有数据结构。
- 依赖影响：优先复用现有 Next.js、TypeScript、Tailwind CSS、shadcn/ui 和 lucide-react，不新增运行时依赖。
- 对现有功能的影响：工作台、事件日志、世界运行记录和创建世界流程保持现状；故事章节列表页作为独立页面新增，不应破坏现有根路径工作台。
- 需要协调的团队：前端团队负责页面、路由、mock 数据和响应式实现；产品/叙事设计负责确认章节字段、摘要长度和后续阅读页内容结构；后端/Agent 团队暂无实现依赖，但需要知晓该页面后续会承接真实故事生成结果。
- 回滚计划：删除新增故事章节列表页路由、组件和 mock 数据；如新增了入口链接则恢复为原有占位或移除入口。由于首版不写入数据、不新增接口，回滚不会影响既有世界运行记录或事件展示。
