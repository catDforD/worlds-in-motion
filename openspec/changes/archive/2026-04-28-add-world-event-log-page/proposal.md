## Why

当前工作台只能展示少量近期事件，且本地事件记录只包含标题、摘要和类型，用户无法系统性查看“这个世界到底发生过什么”。事件日志是后续故事生成、编年史整理和角色传记生成的事实底稿，现在需要把事件从摘要卡片升级为可浏览、可追溯、可扩展的世界事件账本。

## What Changes

- 新增事件日志页面，展示 active world 下的完整事件列表，默认按时间或创建时间倒序排列。
- 在事件列表中展示时间、事件标题、参与角色、地点、事件影响和摘要，帮助用户快速扫描世界进展。
- 提供事件详情视图，展示事件详情、参与角色、地点、影响、事件类型和重要性。
- 扩展本地 `WorldRuntimeEvent` 事件记录字段，支持参与角色、地点、影响、详情和重要性，并兼容旧事件数据。
- 将工作台“近期事件”的查看全部或事件入口连接到完整事件日志页面。
- 保持首版边界：仅使用前端本地状态，不接入后端持久化，不触发 Agent、角色行动或叙事生成。

## Capabilities

### New Capabilities
- `world-event-log-page`: 定义 active world 事件日志页面的路由、列表、详情、排序、空状态和工作台入口行为。

### Modified Capabilities
- `world-event-ledger`: 扩展本地世界运行事件的数据契约，使事件能够作为后续故事生成的结构化事实底稿。

## Impact

- 影响代码：
  - `src/types/world-runtime.ts`：扩展 `WorldRuntimeEvent` 类型和事件重要性类型。
  - `src/lib/world-runtime.ts`：扩展事件解析、默认回退、创建事件和倒序读取逻辑，并保持旧事件兼容。
  - `src/components/dashboard/ink-dashboard.tsx`：让事件入口进入完整事件日志页，并在轻量记录入口中保存新增字段或使用可读回退。
  - 新增事件日志页面相关路由和组件，例如 `src/app/events/page.tsx` 或等价路径，以及必要的事件日志组件。
  - `src/app/globals.css`：补充事件日志页面样式，保持现有水墨工作台视觉语言。
- API 影响：无新增公共 API，无后端接口变更。
- 数据影响：浏览器本地 `worlds-in-motion.world-runtime-state.v1` 中的 `events` 项将增加可选结构化字段；旧事件必须继续可解析和可展示。
- 依赖影响：优先复用现有 Next.js、TypeScript、Tailwind CSS、shadcn/ui 和 lucide-react，不新增运行时依赖。
- 对现有功能的影响：工作台近期事件、统计概览和本地运行记录继续可用；旧事件缺少参与角色、地点、影响或详情时，页面必须使用空值、摘要或可读占位回退，不得导致渲染失败。
- 需要协调的团队：前端团队负责页面、状态读取和视觉实现；产品/叙事设计负责确认事件字段命名、重要性分级和后续故事生成所需事实粒度；后端/Agent 团队暂无实现依赖，但需要知晓本地事件结构将作为后续接口设计参考。
- 回滚计划：移除新增事件日志页面和入口链接，恢复工作台“近期事件”只展示当前摘要卡片；保留扩展字段的本地数据但旧代码只读取既有字段，不影响既有运行记录展示。
