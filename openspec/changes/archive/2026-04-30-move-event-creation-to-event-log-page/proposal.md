## Why

当前工作台的“近期事件”模块同时承担事件预览和手动录入职责，表单体量过大，破坏世界总览页面的信息节奏；事件数量增加后，用户也需要在专门页面中完成完整浏览、选择详情和新增维护。

本变更将世界页定位为只读总览，将事件页定位为事件账本与维护入口，使“世界正在发生什么”和“维护世界发生过什么”形成清晰边界。

## What Changes

- 移除工作台“近期事件”模块中的内嵌记录事件表单。
- 将工作台“近期事件”改为只读预览，只展示 active world 的最新少量事件，并提供进入事件日志页的入口。
- 在事件日志页新增记录事件入口，复用现有本地 `WorldRuntimeState.events` 保存逻辑和结构化事件字段。
- 在事件日志页设计可控的滑动窗口：完整事件列表在列表区域内部滚动，详情或新建表单区域也必须保持可访问，不让页面因事件数量增加被无限撑长。
- 调整无事件空状态，使用户在事件日志页可以创建第一条事件，工作台只负责提示和跳转。
- 保持首版边界：不调用后端、不触发 Agent、不改变事件存储格式。

## Capabilities

### New Capabilities
- 无

### Modified Capabilities
- `ink-dashboard-ui`: 工作台近期事件从“可手动记录事件”调整为“只读近期事件预览”，并保持进入完整事件日志页的入口。
- `world-event-log-page`: 事件日志页从纯浏览详情扩展为事件维护页面，新增记录事件入口、创建校验、保存后刷新列表和滚动窗口要求。

## Impact

- 影响代码：
  - `src/components/dashboard/ink-dashboard.tsx`：移除近期事件表单状态、提交逻辑和 `onRecordEvent` 传递；保留只读事件列表和入口链接。
  - `src/components/events/world-event-log.tsx`：新增记录事件 UI、表单状态、保存逻辑、保存后选中新事件或刷新列表。
  - `src/app/globals.css`：移除或收敛工作台事件表单样式，新增事件页创建入口、表单/抽屉/详情切换和滚动容器样式。
  - 可能更新 Playwright 验证脚本或新增截图，用于确认工作台不再出现录入表单，事件页可以创建事件且列表可滚动。
- API 和数据影响：不新增远程 API；不改变 `WorldRuntimeEvent` 字段和 localStorage 版本；旧事件继续兼容。
- 对现有功能的影响：工作台仍展示近期事件、事件统计和“查看全部”；用户不再能在工作台直接录入事件，需进入事件日志页维护事件。
- 需要协调的团队：前端团队负责页面职责拆分和响应式交互；产品/叙事设计确认“近期事件预览”和“事件日志维护”的文案与字段优先级；后端/Agent 团队无实现依赖，但需要知晓事件维护仍是前端本地首版。
- 回滚计划：恢复工作台近期事件内嵌记录表单和 `onRecordEvent` 处理，移除事件日志页新增创建入口；由于事件数据结构不变，回滚不会影响已保存事件。
