# ink-dashboard-ui Specification

## Purpose
定义织世录根路径水墨工作台的界面能力，包括桌面三栏布局、古风视觉复刻、后端世界信息展示、响应式可读性和前端验证要求。
## Requirements
### Requirement: 水墨工作台主布局
系统 SHALL 在应用根路径呈现一个复刻参考图结构的织世录水墨工作台界面，并提供进入创建世界页面的入口。

#### Scenario: 桌面端呈现三栏工作台
- **GIVEN** 用户使用宽屏桌面视口访问应用根路径
- **WHEN** 页面加载完成
- **THEN** 系统 MUST 显示左侧深色导航栏、中部主内容区和右侧信息栏，并保持三栏之间边界清晰

#### Scenario: 主内容区模块完整
- **GIVEN** 用户访问应用根路径
- **WHEN** 页面加载完成
- **THEN** 系统 MUST 在主内容区显示顶部世界标题横幅、统计概览、运行状态、近期事件、最新故事章节和活跃角色模块

#### Scenario: 右侧信息栏模块完整
- **GIVEN** 用户访问应用根路径
- **WHEN** 页面加载完成
- **THEN** 系统 MUST 在右侧信息栏显示当前时间、紧张关系、待解秘密和近期趋势模块

#### Scenario: 侧栏提供新建世界入口
- **GIVEN** 用户访问应用根路径
- **WHEN** 左侧深色导航栏渲染完成
- **THEN** 系统 MUST 显示可进入 `/worlds/new` 的“新建世界”入口

### Requirement: 参考图视觉复刻
系统 SHALL 以参考图为视觉目标，复刻古风水墨、宣纸、细线卡片和低饱和界面气质。

#### Scenario: 应用水墨古风主题
- **GIVEN** 工作台页面已加载
- **WHEN** 用户查看页面整体视觉
- **THEN** 系统 MUST 使用宣纸浅底、墨色侧栏、细线边框、低饱和辅助色、中文标题层级和古风图标按钮构成一致主题

#### Scenario: 显示山水横幅和插图区域
- **GIVEN** 工作台页面已加载
- **WHEN** 用户查看顶部世界区域和故事章节区域
- **THEN** 系统 MUST 显示山水横幅视觉和章节插图区域，且图片或替代视觉不得为空白、拉伸失真或遮挡文字

#### Scenario: 显示角色头像
- **GIVEN** 工作台页面已加载
- **WHEN** 用户查看关系、事件和活跃角色模块
- **THEN** 系统 MUST 显示角色头像或头像占位视觉，并保持圆形小头像和角色卡头像的尺寸一致

### Requirement: 后端世界信息展示
系统 SHALL 使用后端世界快照、默认 mock 数据、前端临时创建状态或可读回退填充工作台中的世界、角色、事件和运行信息；角色、关系和统计项 MUST 优先读取后端内容种子；当前时间、运行状态、近期事件和最新故事章节 MUST 优先读取后端世界运行记录；近期事件模块 MUST 作为只读预览展示，不在工作台内提供手动记录事件表单。

#### Scenario: 显示默认世界基本信息
- **GIVEN** 用户直接访问工作台且没有创建世界临时状态
- **WHEN** 顶部横幅渲染完成
- **THEN** 系统 MUST 显示默认世界名“烟雨江南”、世界描述、分类标签和顶部操作按钮

#### Scenario: 显示创建后的世界基本信息
- **GIVEN** 用户从创建世界页面完成创建后进入工作台
- **WHEN** 顶部横幅渲染完成
- **THEN** 系统 MUST 显示用户填写的世界名称、背景简介和世界类型标签

#### Scenario: 显示统计概览
- **GIVEN** 用户访问工作台
- **WHEN** 统计概览渲染完成
- **THEN** 系统 MUST 显示角色、势力、地点、事件和秘密五类统计项，并为每类统计项显示图标、数字和单位

#### Scenario: 统计概览读取后端内容种子
- **GIVEN** 用户已经保存角色、势力、地点或关系种子
- **WHEN** 统计概览渲染完成
- **THEN** 系统 MUST 使用后端角色、势力和地点种子的数量更新对应统计项，并保持事件和秘密统计使用现有默认展示或可读回退

#### Scenario: 显示运行状态
- **GIVEN** 用户访问工作台
- **WHEN** 运行状态模块渲染完成
- **THEN** 系统 MUST 显示当前运行或暂停状态、连续运行天数、进度条、下个时间点和暂停或继续运行操作

#### Scenario: 运行状态读取后端运行记录
- **GIVEN** 后端世界运行记录中存在运行天数和暂停状态
- **WHEN** 运行状态模块渲染完成
- **THEN** 系统 MUST 使用后端运行天数和暂停状态更新运行状态文案与操作按钮

#### Scenario: 暂停运行按钮切换状态
- **GIVEN** 工作台已加载后端世界运行记录
- **WHEN** 用户点击暂停或继续运行操作
- **THEN** 系统 MUST 切换后端世界运行记录的暂停状态，并立即在运行状态模块中反映结果

#### Scenario: 当前时间读取后端运行记录
- **GIVEN** 后端世界运行记录中存在当前世界时间
- **WHEN** 右侧信息栏当前时间模块渲染完成
- **THEN** 系统 MUST 显示后端世界运行记录中的当前世界时间，而不是固定 mock 时间

#### Scenario: 显示叙事内容
- **GIVEN** 用户访问工作台
- **WHEN** 事件和故事模块渲染完成
- **THEN** 系统 MUST 显示近期事件预览列表、最新故事章节标题、章节摘要、章节标签、分页点和继续阅读操作

#### Scenario: 叙事内容读取后端运行记录
- **GIVEN** 后端世界运行记录中存在事件列表或最新章节
- **WHEN** 近期事件和最新故事章节模块渲染完成
- **THEN** 系统 MUST 优先使用后端事件列表和最新章节展示叙事内容，并且近期事件 MUST 以最新事件优先展示

#### Scenario: 近期事件为只读预览
- **GIVEN** 用户正在查看工作台近期事件模块
- **WHEN** 近期事件模块渲染完成
- **THEN** 系统 MUST NOT 显示记录事件表单、事件标题输入框、摘要输入框、类型选择框、重要性选择框或提交按钮

#### Scenario: 近期事件入口进入事件日志页
- **GIVEN** 用户正在查看工作台近期事件模块
- **WHEN** 用户点击“查看全部”或等价事件入口
- **THEN** 系统 MUST 导航到事件日志页面，由事件日志页负责完整浏览和手动记录事件

#### Scenario: 近期事件数量受限
- **GIVEN** 后端世界运行记录中存在多条事件
- **WHEN** 工作台近期事件模块渲染完成
- **THEN** 系统 MUST 只展示最新少量事件作为预览，并且 MUST NOT 因完整事件数量增加而无限拉长工作台主内容区

#### Scenario: 推进一日入口可用
- **GIVEN** 用户正在查看工作台运行状态或当前时间模块
- **WHEN** 用户执行“推进一日”
- **THEN** 系统 MUST 更新后端世界运行记录中的当前世界时间和运行天数，并刷新工作台展示

#### Scenario: 显示角色与关系信息
- **GIVEN** 用户访问工作台
- **WHEN** 角色与右侧栏模块渲染完成
- **THEN** 系统 MUST 显示活跃角色卡、紧张关系条目、待解秘密优先级和近期趋势方向

#### Scenario: 角色模块读取后端角色种子
- **GIVEN** 用户已经保存一个或多个角色种子
- **WHEN** 活跃角色模块渲染完成
- **THEN** 系统 MUST 使用后端角色种子的姓名、身份、目标和状态展示角色卡，并为缺失头像使用现有头像占位或默认视觉

#### Scenario: 紧张关系读取后端关系种子
- **GIVEN** 用户已经保存一个或多个关系种子
- **WHEN** 紧张关系模块渲染完成
- **THEN** 系统 MUST 使用后端关系种子的 A、B、关系描述、紧张度和备注展示关系条目

#### Scenario: 无后端内容种子时保持默认展示
- **GIVEN** 用户没有保存任何内容种子
- **WHEN** 工作台角色、关系和统计模块渲染完成
- **THEN** 系统 MUST 继续使用现有默认 mock 数据或可读占位内容，而不是显示空白模块或报错

#### Scenario: 无后端运行记录时保持默认展示
- **GIVEN** 用户没有保存任何后端世界运行记录
- **WHEN** 工作台运行状态、当前时间、近期事件和最新故事章节模块渲染完成
- **THEN** 系统 MUST 继续使用现有默认 mock 数据或可读占位内容，而不是显示空白模块或报错

#### Scenario: 后端内容种子不改写运行态模块
- **GIVEN** 用户保存了角色、势力、地点或关系种子
- **WHEN** 工作台重新渲染
- **THEN** 系统 MUST NOT 根据这些种子伪造或重写运行状态、当前时间、近期事件、最新故事章节、待解秘密或近期趋势模块

#### Scenario: 后端运行记录更新同步展示
- **GIVEN** 用户在工作台暂停运行、继续运行或推进一日
- **WHEN** 系统更新工作台展示
- **THEN** 系统 MUST 调用后端接口保存对应运行记录，并立即在工作台中反映结果

### Requirement: 响应式与可读性
系统 SHALL 在不同视口下保持工作台内容可读且不发生明显重叠。

#### Scenario: 桌面端无明显溢出
- **GIVEN** 用户使用桌面宽屏视口访问工作台
- **WHEN** 页面首屏渲染完成
- **THEN** 系统 MUST 保持导航、卡片、头像、按钮和主要文字在各自容器内可见，且不得出现明显互相遮挡

#### Scenario: 窄屏端可滚动浏览
- **GIVEN** 用户使用窄屏视口访问工作台
- **WHEN** 页面渲染完成
- **THEN** 系统 MUST 将三栏工作台降级为可滚动布局，并保持导航、主内容和右侧信息模块均可访问

#### Scenario: 长文本受控
- **GIVEN** mock 数据或后端数据中存在较长事件、章节或角色描述
- **WHEN** 对应卡片渲染完成
- **THEN** 系统 MUST 通过换行、截断或限定高度处理长文本，避免撑破卡片或覆盖相邻内容

### Requirement: 前端质量验证
系统 SHALL 提供可验证的实现结果，确保静态工作台不破坏现有工程链路。

#### Scenario: 静态检查通过
- **GIVEN** 工作台界面实现完成
- **WHEN** 开发者运行项目静态检查或类型检查命令
- **THEN** 系统 MUST 在无 TypeScript 和 lint 错误的情况下完成检查

#### Scenario: 浏览器截图验收
- **GIVEN** 开发服务器已启动
- **WHEN** 开发者使用浏览器自动化工具访问根路径并截取桌面和窄屏截图
- **THEN** 系统 MUST 渲染非空工作台，并在截图中显示主要模块、视觉资源和可读文本

### Requirement: 运行一天结果面板
系统 SHALL 在工作台侧边栏展示最近一次“运行一天”的本轮变化结果面板，用于呈现事件、关系、秘密、角色目标和故事草稿五类运行摘要。

#### Scenario: 初始无运行结果时显示空状态
- **GIVEN** active world 没有最近一次运行结果
- **WHEN** 用户访问工作台
- **THEN** 系统 MUST 在侧边栏结果面板显示可读空状态，并且 MUST NOT 显示伪造的非零统计

#### Scenario: 点击运行一天后展示本轮统计
- **GIVEN** 工作台已加载 active world 的后端运行记录
- **WHEN** 用户点击“运行一天”
- **THEN** 系统 MUST 更新当前世界时间和运行天数，并在结果面板展示本轮发生事件数、关系变化数、秘密发现数、角色目标改变数和故事草稿生成数

#### Scenario: 结果面板使用明确分类文案
- **GIVEN** 最近一次运行结果包含五类统计
- **WHEN** 结果面板渲染完成
- **THEN** 系统 MUST 使用类似“今日发生了 5 件事”“2 段关系发生变化”“1 个秘密被发现”“1 个角色目标改变”“生成了 1 段故事草稿”的分类文案展示结果

#### Scenario: 展示本轮变化明细
- **GIVEN** 最近一次运行结果中某个分类包含明细文本
- **WHEN** 用户查看结果面板
- **THEN** 系统 MUST 展示该分类的简短明细，并限制长文本展示，避免撑破侧边栏布局

#### Scenario: 刷新后恢复最近运行结果
- **GIVEN** 用户已经点击“运行一天”并保存了最近一次运行结果
- **WHEN** 用户刷新页面或重新访问工作台
- **THEN** 系统 MUST 从 active world 的后端运行记录恢复并展示最近一次运行结果

#### Scenario: 切换世界后展示对应结果
- **GIVEN** 两个世界分别保存了不同的最近运行结果
- **WHEN** 用户切换 active world
- **THEN** 系统 MUST 在结果面板展示切换后世界对应的最近运行结果

#### Scenario: 旧数据缺少结果字段时安全回退
- **GIVEN** active world 的后端运行记录来自旧版本且缺少最近运行结果字段
- **WHEN** 工作台渲染侧边栏结果面板
- **THEN** 系统 MUST 显示空状态或可读回退，并且 MUST NOT 让工作台渲染失败

#### Scenario: 窄屏端结果面板可读
- **GIVEN** 用户使用窄屏视口访问工作台
- **WHEN** 结果面板渲染完成
- **THEN** 系统 MUST 保持统计、明细、按钮和相邻模块可读，且不得出现明显遮挡或横向溢出

#### Scenario: 运行一天不触发 Agent
- **GIVEN** 用户点击“运行一天”
- **WHEN** 系统更新结果面板
- **THEN** 系统 MUST NOT 触发 Agent 工作流
