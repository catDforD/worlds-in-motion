# world-creation-entry Specification

## Purpose
定义独立的新建世界入口和表单体验，用于在纯前端原型中创建本地世界记录、初始化基础设定，并进入工作台继续编辑和运行该世界。
## Requirements
### Requirement: 独立创建世界页面
系统 SHALL 提供独立的创建世界页面，用于承载用户创建第一个或新的叙事世界。

#### Scenario: 访问创建世界页面
- **GIVEN** 用户访问 `/worlds/new`
- **WHEN** 页面加载完成
- **THEN** 系统 MUST 显示创建世界页面，而不是工作台运行视图

#### Scenario: 显示创建页主要区域
- **GIVEN** 用户访问创建世界页面
- **WHEN** 页面渲染完成
- **THEN** 系统 MUST 显示左侧导航、创建世界表单和世界雏形预览区域

#### Scenario: 维持水墨视觉风格
- **GIVEN** 用户查看创建世界页面
- **WHEN** 页面渲染完成
- **THEN** 系统 MUST 使用与现有工作台一致的水墨、宣纸、细线边框和低饱和视觉风格

### Requirement: 创建世界导航入口
系统 SHALL 在现有工作台侧栏提供进入创建世界页面的明确入口。

#### Scenario: 从工作台进入创建页
- **GIVEN** 用户正在查看现有工作台
- **WHEN** 用户点击侧栏中的“新建世界”入口
- **THEN** 系统 MUST 导航到 `/worlds/new`

#### Scenario: 新建入口不破坏原导航
- **GIVEN** 工作台侧栏已加载
- **WHEN** 系统显示“新建世界”入口
- **THEN** 系统 MUST 保留原有“世界、角色、关系、运行、事件、故事、设置、日志、工具”导航项

### Requirement: 创建世界表单字段
系统 SHALL 在创建世界页面提供首版世界设定表单字段。

#### Scenario: 显示基础设定字段
- **GIVEN** 用户访问创建世界页面
- **WHEN** 表单渲染完成
- **THEN** 系统 MUST 显示“世界名称”和“世界类型”字段

#### Scenario: 显示世界底色字段
- **GIVEN** 用户访问创建世界页面
- **WHEN** 表单渲染完成
- **THEN** 系统 MUST 显示“背景简介”和“叙事风格”字段

#### Scenario: 显示运行种子字段
- **GIVEN** 用户访问创建世界页面
- **WHEN** 表单渲染完成
- **THEN** 系统 MUST 显示“世界规则”和“初始冲突”字段

#### Scenario: 支持长文本输入
- **GIVEN** 用户填写背景简介、世界规则或初始冲突
- **WHEN** 输入内容较长
- **THEN** 系统 MUST 保持输入内容可换行编辑，并避免撑破页面布局或覆盖其他控件

### Requirement: 世界类型卡片选择
系统 SHALL 使用选择卡展示世界类型选项，并允许用户选中其中一个类型。

#### Scenario: 显示内置世界类型
- **GIVEN** 用户访问创建世界页面
- **WHEN** 世界类型区域渲染完成
- **THEN** 系统 MUST 显示“宫廷权谋、魔法学院、末日避难所、小镇悬疑、修仙宗门、自定义”六个选项

#### Scenario: 选中世界类型
- **GIVEN** 用户正在创建世界
- **WHEN** 用户选择一个世界类型卡片
- **THEN** 系统 MUST 将该类型标记为当前选中类型，并在预览区域反映该类型

#### Scenario: 自定义类型展开输入
- **GIVEN** 用户正在创建世界
- **WHEN** 用户选择“自定义”类型
- **THEN** 系统 MUST 显示自定义类型名称输入框

#### Scenario: 自定义类型名称用于预览
- **GIVEN** 用户已选择“自定义”类型
- **WHEN** 用户填写自定义类型名称
- **THEN** 系统 MUST 在预览区域显示用户填写的自定义类型名称，而不是只显示“自定义”

### Requirement: 世界雏形实时预览
系统 SHALL 在创建世界页面提供实时更新的世界雏形预览。

#### Scenario: 预览名称和类型
- **GIVEN** 用户正在填写创建世界表单
- **WHEN** 用户输入世界名称并选择世界类型
- **THEN** 系统 MUST 在预览区域显示当前世界名称和世界类型标签

#### Scenario: 预览叙事风格
- **GIVEN** 用户正在填写创建世界表单
- **WHEN** 用户填写叙事风格
- **THEN** 系统 MUST 在预览区域显示当前叙事风格

#### Scenario: 预览初始冲突摘要
- **GIVEN** 用户正在填写创建世界表单
- **WHEN** 用户填写初始冲突
- **THEN** 系统 MUST 在预览区域显示初始冲突摘要，并通过截断或限定高度避免长文本溢出

#### Scenario: 空字段展示占位信息
- **GIVEN** 用户尚未填写部分表单字段
- **WHEN** 预览区域渲染
- **THEN** 系统 MUST 为缺失内容显示可读占位信息，而不是空白或布局塌陷

### Requirement: 创建后进入工作台
系统 SHALL 在首版创建世界成功后创建本地世界库记录，将该世界设为 active，并进入现有工作台展示该世界基本信息。

#### Scenario: 点击创建世界进入工作台
- **GIVEN** 用户已填写创建世界表单所需信息
- **WHEN** 用户点击“创建世界”
- **THEN** 系统 MUST 创建一条新的本地世界记录，将其 `worldId` 设为 active world ID，并导航到现有工作台视图

#### Scenario: 工作台展示新世界信息
- **GIVEN** 用户从创建世界页面完成创建并进入工作台
- **WHEN** 工作台顶部世界横幅渲染完成
- **THEN** 系统 MUST 显示 active world 对应的世界名称、背景简介和世界类型标签

#### Scenario: 创建新世界不覆盖旧世界
- **GIVEN** 世界库中已经存在一个世界及其设定、内容种子或运行记录
- **WHEN** 用户完成另一个世界的创建
- **THEN** 系统 MUST 保留旧世界记录及其按 `worldId` 作用域保存的状态

#### Scenario: 不触发后端持久化
- **GIVEN** 用户点击“创建世界”
- **WHEN** 系统处理首版创建流程
- **THEN** 系统 MUST NOT 调用后端创建世界接口、写入数据库或触发 Agent 世界运行

### Requirement: 创建页响应式与验证
系统 SHALL 保证创建世界页面在桌面和窄屏视口下可读、可操作，并可通过前端质量验证。

#### Scenario: 桌面端布局稳定
- **GIVEN** 用户使用桌面宽屏视口访问创建世界页面
- **WHEN** 页面首屏渲染完成
- **THEN** 系统 MUST 并排显示表单和预览区域，且主要文本、输入框和按钮不得明显重叠

#### Scenario: 窄屏端可滚动填写
- **GIVEN** 用户使用窄屏视口访问创建世界页面
- **WHEN** 页面渲染完成
- **THEN** 系统 MUST 将表单和预览区域降级为可滚动单列布局，并保持所有字段可访问

#### Scenario: 前端检查通过
- **GIVEN** 创建世界入口实现完成
- **WHEN** 开发者运行项目静态检查或类型检查命令
- **THEN** 系统 MUST 在无 TypeScript 和 lint 错误的情况下完成检查
