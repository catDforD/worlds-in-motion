# world-seed-assets Specification

## Purpose
定义当前世界下角色、势力、地点和关系种子的前端管理能力，包括本地创建、编辑、删除、保存、恢复，以及与工作台统计、活跃角色和紧张关系模块的联动展示。
## Requirements
### Requirement: 世界内容种子管理入口
系统 SHALL 在当前世界工作台中提供管理世界内容种子的入口，并允许用户打开角色、势力、地点和关系种子的编辑界面。

#### Scenario: 从工作台打开内容种子界面
- **GIVEN** 用户正在查看工作台
- **WHEN** 用户点击内容种子或资源相关入口
- **THEN** 系统 MUST 显示世界内容种子编辑界面

#### Scenario: 显示四类种子分区
- **GIVEN** 用户已经打开世界内容种子编辑界面
- **WHEN** 编辑界面渲染完成
- **THEN** 系统 MUST 显示角色种子、势力种子、地点种子和关系种子四类可编辑分区

#### Scenario: 无本地种子时显示空状态
- **GIVEN** 当前浏览器没有保存过世界内容种子
- **WHEN** 用户打开世界内容种子编辑界面
- **THEN** 系统 MUST 为四类种子显示可读空状态或默认空行，且不得导致页面报错

### Requirement: 角色种子维护
系统 SHALL 允许用户创建、编辑和删除角色种子，并维护姓名、身份、目标和状态字段。

#### Scenario: 新增角色种子
- **GIVEN** 用户正在编辑角色种子
- **WHEN** 用户填写姓名、身份、目标和状态并保存
- **THEN** 系统 MUST 将该角色加入当前世界的角色种子列表

#### Scenario: 编辑角色种子
- **GIVEN** 当前世界已经存在角色种子
- **WHEN** 用户修改该角色的姓名、身份、目标或状态并保存
- **THEN** 系统 MUST 使用修改后的字段更新该角色种子

#### Scenario: 删除角色种子
- **GIVEN** 当前世界已经存在角色种子
- **WHEN** 用户执行删除操作
- **THEN** 系统 MUST 从当前世界的角色种子列表中移除该角色

#### Scenario: 角色长文本受控
- **GIVEN** 用户在角色目标或状态中输入较长文本
- **WHEN** 角色种子编辑界面和工作台角色模块渲染
- **THEN** 系统 MUST 通过换行、截断或限定高度保持文本在容器内可读，且不得遮挡相邻内容

### Requirement: 势力和地点种子维护
系统 SHALL 允许用户创建、编辑和删除势力种子与地点种子，并维护首版必要字段。

#### Scenario: 新增势力种子
- **GIVEN** 用户正在编辑势力种子
- **WHEN** 用户填写名称、立场、资源和冲突点并保存
- **THEN** 系统 MUST 将该势力加入当前世界的势力种子列表

#### Scenario: 新增地点种子
- **GIVEN** 用户正在编辑地点种子
- **WHEN** 用户填写名称、类型和重要性并保存
- **THEN** 系统 MUST 将该地点加入当前世界的地点种子列表

#### Scenario: 编辑势力或地点种子
- **GIVEN** 当前世界已经存在势力种子或地点种子
- **WHEN** 用户修改该种子的任一字段并保存
- **THEN** 系统 MUST 使用修改后的字段更新对应种子

#### Scenario: 删除势力或地点种子
- **GIVEN** 当前世界已经存在势力种子或地点种子
- **WHEN** 用户执行删除操作
- **THEN** 系统 MUST 从对应种子列表中移除该条目

### Requirement: 关系种子维护
系统 SHALL 允许用户创建、编辑和删除关系种子，并维护 A 与 B 的关系、紧张度和备注。

#### Scenario: 新增关系种子
- **GIVEN** 用户正在编辑关系种子
- **WHEN** 用户填写 A、B、关系描述、紧张度和备注并保存
- **THEN** 系统 MUST 将该关系加入当前世界的关系种子列表

#### Scenario: 编辑关系紧张度
- **GIVEN** 当前世界已经存在关系种子
- **WHEN** 用户修改该关系的紧张度并保存
- **THEN** 系统 MUST 在关系种子列表和工作台紧张关系展示中使用新的紧张度

#### Scenario: 关系参与方可自由输入
- **GIVEN** 用户正在新增关系种子
- **WHEN** 用户输入尚未存在于角色、势力或地点种子中的参与方名称
- **THEN** 系统 MUST 允许保存该关系，并按用户输入的名称展示

#### Scenario: 紧张度边界处理
- **GIVEN** 本地关系种子中的紧张度缺失或超出允许范围
- **WHEN** 系统读取该关系种子
- **THEN** 系统 MUST 将紧张度回退或限制到可展示范围内，且不得阻断页面渲染

### Requirement: 内容种子本地保存与恢复
系统 SHALL 将世界内容种子保存到浏览器本地状态，并在刷新后恢复。

#### Scenario: 保存内容种子到本地状态
- **GIVEN** 用户已经修改角色、势力、地点或关系种子
- **WHEN** 用户点击保存操作
- **THEN** 系统 MUST 将完整内容种子集合保存到浏览器本地状态

#### Scenario: 刷新后恢复内容种子
- **GIVEN** 用户已经保存过世界内容种子
- **WHEN** 用户刷新浏览器并重新进入工作台
- **THEN** 系统 MUST 从浏览器本地状态恢复角色、势力、地点和关系种子

#### Scenario: 本地种子数据损坏时回退
- **GIVEN** 浏览器本地状态中的内容种子数据无法解析或字段类型不符合预期
- **WHEN** 用户进入工作台或打开内容种子编辑界面
- **THEN** 系统 MUST 回退到空种子或默认 mock 展示，并且不得阻断页面渲染

#### Scenario: 保存后不调用后端
- **GIVEN** 用户点击保存内容种子
- **WHEN** 系统处理保存操作
- **THEN** 系统 MUST NOT 调用后端接口、写入数据库或触发 Agent 世界运行

### Requirement: 内容种子界面响应式与验证
系统 SHALL 保证内容种子编辑界面在桌面和窄屏视口下可读、可操作，并可通过前端质量验证。

#### Scenario: 桌面端内容种子界面稳定
- **GIVEN** 用户使用桌面宽屏视口打开内容种子编辑界面
- **WHEN** 编辑界面渲染完成
- **THEN** 系统 MUST 保持分区、字段、列表、按钮和主要文本在各自容器内可见，且不得明显互相遮挡

#### Scenario: 窄屏端内容种子界面可滚动
- **GIVEN** 用户使用窄屏视口打开内容种子编辑界面
- **WHEN** 编辑界面渲染完成
- **THEN** 系统 MUST 以可滚动布局展示四类种子和保存操作，并保持所有字段可访问

#### Scenario: 前端检查通过
- **GIVEN** 内容种子实现完成
- **WHEN** 开发者运行项目静态检查或类型检查命令
- **THEN** 系统 MUST 在无 TypeScript 和 lint 错误的情况下完成检查
