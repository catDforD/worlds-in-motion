## Purpose

定义工作台中当前世界设定的前端编辑能力，包括打开设定编辑界面、维护核心设定字段、本地保存与恢复、保存后联动工作台展示，以及兼容创建世界流程产生的本地状态。
## Requirements
### Requirement: 工作台世界设定编辑入口
系统 SHALL 在工作台中提供当前世界设定的编辑入口，并允许用户从工作台打开设定编辑界面。

#### Scenario: 从工作台打开设定编辑界面
- **GIVEN** 用户正在查看工作台
- **WHEN** 用户点击顶部横幅中的“世界设定”操作
- **THEN** 系统 MUST 显示当前世界设定编辑界面

#### Scenario: 编辑界面保持当前世界上下文
- **GIVEN** 用户已经创建或保存过当前世界设定
- **WHEN** 用户打开世界设定编辑界面
- **THEN** 系统 MUST 使用当前世界设定填充编辑字段，而不是展示空白表单

#### Scenario: 无本地设定时打开编辑界面
- **GIVEN** 用户没有创建或保存过本地世界设定
- **WHEN** 用户打开世界设定编辑界面
- **THEN** 系统 MUST 使用默认工作台世界信息或可读占位内容填充编辑字段，且不得导致界面报错

### Requirement: 核心设定字段编辑
系统 SHALL 允许用户编辑世界背景、世界规则、风格偏好、禁止事项和核心矛盾五类核心设定。

#### Scenario: 显示核心设定字段
- **GIVEN** 用户打开世界设定编辑界面
- **WHEN** 编辑界面渲染完成
- **THEN** 系统 MUST 显示“世界背景”“世界规则”“风格偏好”“禁止事项”“核心矛盾”字段

#### Scenario: 编辑长文本设定
- **GIVEN** 用户正在编辑世界背景、世界规则、禁止事项或核心矛盾
- **WHEN** 用户输入多行长文本
- **THEN** 系统 MUST 保持文本可换行编辑，并避免输入内容撑破编辑界面或遮挡保存操作

#### Scenario: 保存空白可选字段
- **GIVEN** 用户清空风格偏好、禁止事项或核心矛盾字段
- **WHEN** 用户保存世界设定
- **THEN** 系统 MUST 保存用户的空白输入，并在展示区域使用可读占位或隐藏空标签处理空值

### Requirement: 本地保存与恢复
系统 SHALL 将世界设定按 active `worldId` 保存到浏览器本地状态，并在刷新或切换世界后恢复对应世界的设定。

#### Scenario: 保存世界设定到本地状态
- **GIVEN** 用户已经修改 active world 的世界设定字段
- **WHEN** 用户点击保存操作
- **THEN** 系统 MUST 将世界设定保存到 active `worldId` 对应的浏览器本地状态

#### Scenario: 保存后不调用后端
- **GIVEN** 用户点击保存世界设定
- **WHEN** 系统处理保存操作
- **THEN** 系统 MUST NOT 调用后端世界接口、写入数据库或触发 Agent 世界运行

#### Scenario: 刷新后恢复世界设定
- **GIVEN** 用户已经保存过 active world 的世界设定
- **WHEN** 用户刷新浏览器并重新进入工作台
- **THEN** 系统 MUST 从浏览器本地状态恢复 active `worldId` 对应的已保存世界设定

#### Scenario: 切换世界后恢复对应设定
- **GIVEN** 两个世界分别保存了不同的世界设定
- **WHEN** 用户切换 active world
- **THEN** 系统 MUST 在工作台和设定编辑界面展示切换后 `worldId` 对应的设定

#### Scenario: 本地数据损坏时回退
- **GIVEN** 浏览器本地状态中的世界设定数据无法解析或字段类型不符合预期
- **WHEN** 用户进入工作台或打开世界设定编辑界面
- **THEN** 系统 MUST 回退到 active world 的基础信息、默认工作台世界信息或可读占位内容，并且不得阻断页面渲染

### Requirement: 保存后工作台联动展示
系统 SHALL 在用户保存世界设定后立即更新工作台中可展示的当前世界信息。

#### Scenario: 保存后更新顶部横幅
- **GIVEN** 用户在世界设定编辑界面修改了世界背景或风格偏好
- **WHEN** 用户保存世界设定
- **THEN** 工作台顶部横幅 MUST 立即显示更新后的世界背景摘要和风格标签

#### Scenario: 同页保存无需刷新
- **GIVEN** 用户在工作台中打开世界设定编辑界面
- **WHEN** 用户保存修改
- **THEN** 系统 MUST 在不刷新页面的情况下更新工作台展示

#### Scenario: 不改写运行态 mock 模块
- **GIVEN** 用户保存世界规则、禁止事项或核心矛盾
- **WHEN** 工作台重新渲染
- **THEN** 系统 MUST NOT 根据这些字段伪造或重写运行状态、近期事件、角色关系、待解秘密或趋势模块

### Requirement: 创建流程兼容设定编辑
系统 SHALL 兼容创建世界流程产生的前端状态，并让 active world 可进入设定编辑。

#### Scenario: 创建后初始化可编辑设定
- **GIVEN** 用户在 `/worlds/new` 完成创建世界
- **WHEN** 系统进入工作台
- **THEN** 系统 MUST 让“世界设定”编辑界面显示 active world 创建时填写的背景、规则、风格和初始冲突对应内容

#### Scenario: 兼容旧创建临时状态
- **GIVEN** 浏览器中存在旧版创建世界临时状态但不存在新世界库
- **WHEN** 用户进入工作台
- **THEN** 系统 MUST 将旧版创建状态迁移为一个 active world，并可在打开编辑界面时映射为可编辑世界设定

### Requirement: 编辑界面响应式与验证
系统 SHALL 保证世界设定编辑界面在桌面和窄屏视口下可读、可操作，并可通过前端质量验证。

#### Scenario: 桌面端编辑界面稳定
- **GIVEN** 用户使用桌面宽屏视口打开世界设定编辑界面
- **WHEN** 编辑界面渲染完成
- **THEN** 系统 MUST 保持字段、按钮和主要文本在各自容器内可见，且不得明显互相遮挡

#### Scenario: 窄屏端编辑界面可滚动
- **GIVEN** 用户使用窄屏视口打开世界设定编辑界面
- **WHEN** 编辑界面渲染完成
- **THEN** 系统 MUST 以可滚动布局展示所有字段和保存操作，并保持所有字段可访问

#### Scenario: 前端检查通过
- **GIVEN** 世界设定编辑实现完成
- **WHEN** 开发者运行项目静态检查或类型检查命令
- **THEN** 系统 MUST 在无 TypeScript 和 lint 错误的情况下完成检查
